# /feature

Implementar un feature o un cambio acotado.

## Pasos

1. Si faltan requisitos o criterios de aceptación, preguntar antes de empezar.
2. Si hay mockups o specs de API en `docs/ref/`, revisarlos.
3. Proponer un plan con los archivos afectados y esperar confirmación antes de codificar.
4. Implementar siguiendo el flujo de nuevo feature en `docs/STANDARDS.md`. Usar el feature `transactions` como referencia cuando aplique.
5. Correr `npm run lint` y `npm test`.
6. Si el cambio afecta un recorrido de usuario, señalar si hace falta flujo Maestro.
7. Resumir qué se hizo y qué conviene validar en dispositivo o emulador.
