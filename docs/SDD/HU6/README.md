## Criterios de aceptación
<ul>
<li><b>Dado</b> que el cliente ha iniciado sesión, <b>cuando</b> se carga el Home, <b>entonces</b> se muestra la opción “Transferencia”. </li>
<li><b>Dado</b> que el cliente está en el Home, <b>cuando</b> selecciona la opción “Transferencia”, <b>entonces</b> el sistema navega al módulo de Transferencias. </li>
<li><b>Dado</b> que el cliente ingresa al módulo de Transferencias, <b>cuando</b> se presenta la vista inicial, <b>entonces</b> se muestran las opciones de tipo de transferencia: “Entre mis cuentas”, “Terceros (Bolivariano)” y “Otros bancos”. </li>
<li><b>Dado</b> que el cliente está en el módulo de Transferencias, <b>cuando</b> se muestra la vista inicial, <b>entonces</b> se presenta la sección “Contactos favoritos”. </li>
<li><b>Dado</b> que el cliente está en “Contactos favoritos”, <b>cuando</b> no existen contactos favoritos configurados, <b>entonces</b> se muestra el mensaje “No tienes contactos favoritos” y una acción para continuar al formulario sin seleccionar un favorito. </li>
<li><b>Dado</b> que el cliente visualiza “Contactos favoritos”, <b>cuando</b> selecciona un contacto favorito, <b>entonces</b> el sistema precarga el destinatario (campo “Para”) en el formulario de transferencia. </li>
<li><b>Dado</b> que el cliente seleccionó un tipo de transferencia, <b>cuando</b> se muestra el formulario, <b>entonces</b> se visualizan los campos: “Para” (beneficiario), “Desde” (cuenta origen), “Monto a transferir” y “Concepto”. </li>
<li><b>Dado</b> que el cliente está en el formulario, <b>cuando</b> intenta continuar sin seleccionar “Para” (beneficiario), <b>entonces</b> el sistema marca el campo como inválido y muestra un mensaje indicando que es obligatorio. </li>
<li><b>Dado</b> que el cliente está en el formulario, <b>cuando</b> intenta continuar sin seleccionar “Desde” (cuenta origen), <b>entonces</b> el sistema marca el campo como inválido y muestra un mensaje indicando que es obligatorio. </li>
<li><b>Dado</b> que el cliente está en el formulario, <b>cuando</b> el “Monto a transferir” está vacío o es menor o igual a 0, <b>entonces</b> el campo se considera inválido y no se permite continuar. </li>
<li><b>Dado</b> que el cliente ingresa un “Monto a transferir”, <b>cuando</b> el monto excede el límite máximo permitido por la regla de negocio vigente o es menor al mínimo permitido, <b>entonces</b> se muestra un mensaje de validación indicando el/los límites aplicables y el campo se considera inválido. </li>
<li><b>Dado</b> que el cliente ingresa un “Monto a transferir” válido, <b>cuando</b> el sistema evalúa las condiciones de la transferencia, <b>entonces</b> se calcula y muestra la comisión aplicable (o “Sin cargo”) antes de continuar. </li>
<li><b>Dado</b> que el cliente está en el formulario, <b>cuando</b> el “Concepto” está vacío, <b>entonces</b> el campo se considera inválido y no se permite continuar. </li>
<li><b>Dado</b> que el cliente ingresa un “Concepto”, <b>cuando</b> excede la longitud máxima permitida por la regla de negocio vigente, <b>entonces</b> se muestra un mensaje de validación y el campo se considera inválido. </li>
<li><b>Dado</b> que el cliente está en el formulario, <b>cuando</b> se muestran los datos informativos, <b>entonces</b> se visualizan de forma clara: comisión (si aplica), monto mínimo, monto máximo y cualquier condición relevante del tipo de transferencia seleccionado. </li>
<li><b>Dado</b> que el cliente está en el formulario, <b>cuando</b> existe al menos un campo obligatorio vacío o inválido (“Para”, “Desde”, “Monto a transferir” o “Concepto”), <b>entonces</b> el botón “Continuar” permanece deshabilitado. </li>
<li><b>Dado</b> que el cliente completó y validó “Para”, “Desde”, “Monto a transferir” y “Concepto”, <b>cuando</b> todos los campos son válidos, <b>entonces</b> el botón “Continuar” se habilita. </li>
<li><b>Dado</b> que el cliente presiona “Continuar” con datos válidos, <b>cuando</b> el sistema procesa la solicitud, <b>entonces</b> se navega a la pantalla de resumen/confirmación. </li>
<li><b>Dado</b> que el cliente está en la pantalla de resumen/confirmación, <b>cuando</b> se muestra el detalle, <b>entonces</b> se visualiza: destinatario (Para), cuenta origen (Desde), monto a enviar, comisión (con o sin cargo), total a debitar (monto + comisión), fecha y concepto. </li>
<li><b>Dado</b> que el cliente está en la pantalla de resumen/confirmación, <b>cuando</b> selecciona “Modificar”, <b>entonces</b> el sistema regresa al formulario conservando los valores previamente ingresados para edición. </li>
<li><b>Dado</b> que el cliente está en la pantalla de resumen/confirmación, <b>cuando</b> selecciona “Confirmar”, <b>entonces</b> el sistema navega a la pantalla de ingreso del código de factor de seguridad. </li>
<li><b>Dado</b> que el cliente está en la pantalla de factor de seguridad, <b>cuando</b> se presenta la vista, <b>entonces</b> se muestran: monto, destinatario, cuenta destinatario y un campo para ingresar una clave de 6 dígitos con teclado numérico en pantalla. </li>
<li><b>Dado</b> que el cliente está en la pantalla de factor de seguridad, <b>cuando</b> ingresa menos de 6 dígitos, <b>entonces</b> el sistema no permite confirmar la autorización. </li>
<li><b>Dado</b> que el cliente está en la pantalla de factor de seguridad, <b>cuando</b> presiona “Cancelar”, <b>entonces</b> el sistema retorna al formulario para permitir edición. </li>
<li><b>Dado</b> que el cliente ingresó correctamente la clave de 6 dígitos, <b>cuando</b> presiona “Confirmar autorización”, <b>entonces</b> el sistema procesa la transferencia y muestra la pantalla de éxito. </li>
<li><b>Dado</b> que la transferencia fue procesada exitosamente, <b>cuando</b> se muestra la pantalla de éxito, <b>entonces</b> se visualiza “Transferencia Exitosa” y el comprobante con: monto, destinatario, cuenta destinatario, número de referencia, fecha y hora, concepto y origen. </li>
<li><b>Dado</b> que el cliente está en la pantalla de éxito, <b>cuando</b> selecciona “Compartir”, <b>entonces</b> el sistema permite compartir el comprobante mediante los mecanismos disponibles del dispositivo. </li>
<li><b>Dado</b> que el cliente está en la pantalla de éxito, <b>cuando</b> selecciona “Descargar”, <b>entonces</b> el sistema descarga/guarda el comprobante en el dispositivo según el comportamiento estándar de la app. </li>
<li><b>Dado</b> que el cliente está en la pantalla de éxito, <b>cuando</b> selecciona “Nueva transferencia”, <b>entonces</b> el sistema navega al formulario de transferencia listo para una nueva operación. </li>
<li><b>Dado</b> que el cliente está en la pantalla de éxito, <b>cuando</b> selecciona “Ir al inicio”, <b>entonces</b> el sistema navega al Home. </li>
</ul>
