# Pantalla de inicio de sesión

## Descripción
<div>Yo como Banco Bolivariano,&nbsp; </div><div>quiero una pantalla de inicio de sesión donde mis clientes ingresen con usuario y contraseña,&nbsp; </div><div>para permitir el acceso a la aplicación móvil de manera rápida y segura. </div><div>0) Al instalar la aplicación debe mostra un splash con el logo del BBOL. </div><div><div style="box-sizing:border-box;">1) Crear una pantalla que me permita ingresar un usuario y contraseña válida. </div><div style="box-sizing:border-box;">2) Debe permitir poder realizar login a través del Biométrico. </div><div style="box-sizing:border-box;">3) Debe mostrar el botón de &quot;Iniciar sesión&quot; </div><div style="box-sizing:border-box;">4) Debe mostrar la opción de &quot;Contáctanos&quot;&nbsp; </div><div style="box-sizing:border-box;">5) Al ingresar un usuario y contraseña y presionar el botón &quot;Iniciar sesión&quot;; la aplicación debe mostrar una pantalla para el ingreso del OTP 6 dígitos/caracteres que debe enviar al correo registrado. </div><div style="box-sizing:border-box;"><br> </div><div style="box-sizing:border-box;"><b>Prototipo:</b> </div><div style="box-sizing:border-box;"><a href="https://www.figma.com/design/1WEEEIsZQXjPjUhjY2n1Oh/Prueba-Hackathon?node-id=74-462&amp;p=f&amp;t=GL4eUdCEH9bKuLWB-0">Prueba Hackathon – Figma</a><br style="box-sizing:border-box;"> </div><br><img src="https://dev.azure.com/BayteqDev/a618cca1-fc1f-4133-b148-7ffb6919218e/_apis/wit/attachments/05b0ab2d-9021-4eab-8d0d-1350a00dd689?fileName=image.png" alt=Image><br> </div><div><br> </div>

## Criterios de aceptación
<div><ul><li>- Dado que el cliente accede al módulo de autenticación, cuando se carga la pantalla de inicio de sesión, entonces se muestran los campos &quot;Usuario &quot; y &quot;Contraseña&quot;.
</li><li>
- Dado que el cliente ingresa un valor en &quot;usuario&quot;, el mismo debe tener un mínimo de 8 caracteres y máximo 15 caracteres; no debe permitir el ingreso de caracteres especiales. </li><li>
- Dado que el cliente está en la pantalla de inicio de sesión, cuando visualiza el campo &quot;Contraseña&quot;, entonces el contenido se muestra enmascarado (tipo password).
</li><li><span style="display:inline !important;">Dado que el cliente está en la pantalla de inicio de sesión; cuando se ingresa la contraseña la misma deben tener: mínimo 6 caracteres, máximo 15 caracteres; debe tener por lo menos un caracter especial, un número y una letra Mayúscula.&nbsp;</span> </li><li>- Dado que el cliente está en la pantalla de inicio de sesión, cuando se ha ingresado un usuario y contraseñan válido se debe habilitar el botón de &quot;Iniciar sesión&quot;. </li><li>- Al ingresar el usuario y contraseña válida debe solicitar la<span style="display:inline !important;">&nbsp;OTP 6 dígitos/caracteres que debe enviar al correo registrado.</span> </li> </ul> </div><div> </div><div> </div><div><ul><li><span style="display:inline !important;">Dado que el cliente accede al inicio de sesión, cuando tenga 3 intentos fallidos, entonces se debe mostrar un mensaje de &quot;Usuario bloqueado por demasiados intentos&quot;</span> </li> </ul><span style="display:inline !important;"></span> </div><div><br> </div><div><br> </div>
