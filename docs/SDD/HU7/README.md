# Consulta de Transacciones

## Descripción
<div>Como usuario del banco Bolivariano,&nbsp; </div><div>quiero consultar mis movimientos desde una pantalla general con filtros y acceder al detalle de un movimiento, para controlar mis transacciones, obtener comprobantes y reportar actividades sospechosas.&nbsp; </div><div><br> </div><div><b>&nbsp;Flujo:&nbsp;</b> </div><div>1) Pantalla 1 (General): el usuario selecciona una cuenta y aplica filtros (nombre, rango de fechas, tipo, rango de monto) para ver una lista de movimientos ordenada por fecha y agrupada por periodos (Hoy, Ayer, Semana anterior, etc.). Cada movimiento muestra ícono según tipo, nombre, fecha/hora, monto (débito o crédito), y posición consolidada.&nbsp; </div><div>2) Al seleccionar un movimiento, navega a Pantalla 2 (Detalle): se presenta la información completa de la transferencia (monto, nombre, fecha/hora, tipo, número de referencia, canal, cuenta de origen) con acciones para compartir y descargar el comprobante.&nbsp; </div><div>3) Al final del detalle se muestra la opción de reportar el movimiento como sospechoso, mostrando un mensaje final de confirmación. </div>

## Criterios de aceptación
<ul><li><strong>Dado</strong> que el usuario ingresa al módulo de Movimientos, <strong>cuando</strong> se muestra la Pantalla 1 (General), <strong>entonces</strong> debe existir un selector de cuenta y los filtros: nombre (texto), fecha (rango desde/hasta), tipo (lista de tipos) y monto (rango mínimo/máximo). </li><li><strong>Dado</strong> que el usuario selecciona una cuenta, <strong>cuando</strong> se carga la lista, <strong>entonces</strong> se muestran únicamente movimientos asociados a la cuenta seleccionada. </li><li><strong>Dado</strong> que el usuario ingresa un valor en el filtro por nombre, <strong>cuando</strong> aplica el filtro, <strong>entonces</strong> la lista incluye solo movimientos cuyo nombre/descripcion contenga el texto ingresado (no sensible a mayúsculas/minúsculas). </li><li><strong>Dado</strong> que el usuario selecciona un rango de fechas (desde/hasta), <strong>cuando</strong> aplica el filtro, <strong>entonces</strong> la lista incluye solo movimientos dentro del rango inclusivo. </li><li><strong>Dado</strong> que el usuario selecciona un tipo de movimiento, <strong>cuando</strong> aplica el filtro, <strong>entonces</strong> la lista incluye solo movimientos del tipo seleccionado. </li><li><strong>Dado</strong> que el usuario define un rango de monto (mínimo/máximo), <strong>cuando</strong> aplica el filtro, <strong>entonces</strong> la lista incluye solo movimientos cuyo monto esté dentro del rango inclusivo. </li><li><strong>Dado</strong> que el usuario establece un rango de monto, <strong>cuando</strong> el mínimo sea mayor que el máximo, <strong>entonces</strong> el sistema debe bloquear la aplicación del filtro y mostrar un mensaje de validación indicando que el mínimo no puede ser mayor que el máximo. </li><li><strong>Dado</strong> que el usuario establece un rango de fechas, <strong>cuando</strong> la fecha desde sea posterior a la fecha hasta, <strong>entonces</strong> el sistema debe bloquear la aplicación del filtro y mostrar un mensaje de validación indicando que la fecha inicial no puede ser mayor que la fecha final. </li><li><strong>Dado</strong> que existen movimientos para la cuenta y filtros aplicados, <strong>cuando</strong> se renderiza la lista, <strong>entonces</strong> debe estar ordenada de forma descendente por fecha y hora (más recientes primero). </li><li><strong>Dado</strong> que se muestra la lista, <strong>cuando</strong> existen movimientos en distintos periodos, <strong>entonces</strong> deben agruparse por encabezados de periodo (al menos: Hoy, Ayer, Semana anterior, Anteriores) según la fecha del movimiento. </li><li><strong>Dado</strong> que se muestra un ítem de movimiento en la lista, <strong>cuando</strong> se presenta al usuario, <strong>entonces</strong> debe incluir: ícono según tipo, nombre, fecha y hora, monto con indicador de débito o crédito, y la posición consolidada. </li><li><strong>Dado</strong> que el movimiento es de tipo débito, <strong>cuando</strong> se muestra el monto, <strong>entonces</strong> debe identificarse como debitado (ej. signo negativo o etiqueta “Débito”) de forma consistente en toda la lista. </li><li><strong>Dado</strong> que el movimiento es de tipo crédito, <strong>cuando</strong> se muestra el monto, <strong>entonces</strong> debe identificarse como acreditado (ej. signo positivo o etiqueta “Crédito”) de forma consistente en toda la lista. </li><li><strong>Dado</strong> que no existan movimientos para la cuenta/filtros, <strong>cuando</strong> se muestra la Pantalla 1, <strong>entonces</strong> debe mostrarse un estado vacío indicando que no hay resultados y permitir ajustar filtros. </li><li><strong>Dado</strong> que el usuario selecciona (toca/clic) un movimiento en la lista, <strong>cuando</strong> navega a la Pantalla 2 (Detalle), <strong>entonces</strong> se debe mostrar la información: monto, nombre, fecha y hora, tipo, número de referencia, canal y cuenta de origen. </li><li><strong>Dado</strong> que el usuario está en la Pantalla 2, <strong>cuando</strong> presiona “Compartir”, <strong>entonces</strong> el sistema debe abrir el mecanismo de compartición con el comprobante del movimiento. </li><li><strong>Dado</strong> que el usuario está en la Pantalla 2, <strong>cuando</strong> presiona “Descargar comprobante”, <strong>entonces</strong> el sistema debe descargar/guardar el comprobante del movimiento y notificar el resultado (éxito o fallo). </li><li><strong>Dado</strong> que el usuario está en la Pantalla 2, <strong>cuando</strong> visualiza la parte final de la pantalla, <strong>entonces</strong> debe existir la opción “Reportar movimiento” para marcarlo como sospechoso. </li><li><strong>Dado</strong> que el usuario presiona “Reportar movimiento”, <strong>cuando</strong> confirma el reporte, <strong>entonces</strong> el sistema debe registrar el reporte asociado al movimiento (por número de referencia o identificador) y mostrar un mensaje final de confirmación. </li> </ul>
