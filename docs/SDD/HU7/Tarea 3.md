# 3. Enmascarar número de cuenta destino y mostrar cuenta origen en claro

**Prioridad**: Alta

**Historia padre**: #29253

## Descripción
Implementar lógica de presentación: cuenta origen debe mostrarse con tipo y número en claro; cuenta destino debe mostrarse con número enmascarado dejando visibles solo los últimos 3 dígitos. Encapsular la lógica de máscara en un helper reutilizable y cubrir con pruebas unitarias (casos: longitud 3, nulos).
